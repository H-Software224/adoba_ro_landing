# adobaRo GEO 최적화 가이드: 스키마 마크업 & 헤딩 구조

> GEO(Generative Engine Optimization)의 핵심은 "AI가 페이지 구조만 보고도 핵심 정보를 정확히 발췌할 수 있는가"입니다.
> 헤딩은 AI가 콘텐츠를 청크(chunk) 단위로 나누는 기준이 되고, 스키마(JSON-LD)는 그 청크에 의미(가격, 질문/답변, 이벤트 기간 등)를 명시적으로 태깅해주는 역할입니다.

---

## 0. 전 페이지 공통 (Global, `<head>` 또는 레이아웃 컴포넌트에 삽입)

| 스키마 타입 | 용도 | 채울 값 |
|---|---|---|
| `Organization` | 회사 신뢰도 정보를 AI가 인용할 때 참조 | name: 아도바 주식회사, url(⚠️ **메인 도메인 하나로 전 페이지 고정** — 아래 참고), logo, address(서울 구로구 공원로 41, 421호), telephone(02-6952-0109), email(support@adoba.net) |
| `WebSite` | 사이트 전체 식별 + 검색 기능이 있다면 `SearchAction` | name: adobaRo, url(Organization과 동일한 메인 도메인) |

> ⚠️ **`url` 값 관련 주의**: `Organization.url`과 `WebSite.url`은 랜딩페이지마다 바뀌는 값이 아니라 adobaRo 공식 홈페이지의 **대표(루트) 도메인 하나**를 전 페이지에 동일하게 고정 삽입합니다. 페이지별로 달라져야 하는 건 각 페이지 스키마의 `mainEntityOfPage` / 개별 `url` 필드(예: 요금제 페이지의 Product 스키마엔 요금제 페이지 URL, 아티클의 BlogPosting 스키마엔 해당 아티클 URL)이며, 이는 각 섹션에서 별도로 명시하고 있습니다.
| `SiteNavigationElement` | 상단 GNB(기능/요금제/adobaRo 소식/해외 진출 Tip) 구조화 | 4개 메뉴 각각 name+url |

이 3개는 모든 페이지에 반복 삽입(또는 layout에서 1회 정의)하는 게 GEO에서 브랜드 개체(entity) 인식에 유리합니다.

## 1. Main (홈)

**헤딩 구조**
- `H1`: AI로 글로벌 진출 바로 실행! *(페이지당 H1은 1개만 — 현재 히어로 카피가 적절)*
- `H2`: 지금 크리에이터가 겪고 있는 문제, 해답은 글로벌 진출입니다 / 복잡한 해외 진출, adobaRo에선 간편하게 / adobaRo 없이 글로벌 진출한다면? / adobaRo와 글로벌 진출했을 때 / adobaRo로 글로벌 진출한 크리에이터의 실제 후기 / 자주 묻는 질문
- `H3`: 4개 기능 카드명(자동 번역·현지화 / 원클릭 멀티 배포 / 브랜드 광고·협업 / 성과 트래킹), 비교 테이블의 행 헤더(비용/다국어 영상 편집·싱크/대기 시간/영상 1편 기준 총비용), 후기 작성자 3명

**스키마 마크업**
| 우선순위 | 타입 | 위치 | 비고 |
|---|---|---|---|
| ★★★ | `FAQPage` | "자주 묻는 질문" 5문항 | 가장 확실한 GEO 효과. 질문=`Question`, 답변 전체 텍스트=`acceptedAnswer` |
| ★★★ | `Service` (또는 `SoftwareApplication`) | H1 바로 아래 서브텍스트 문단 ("adobaRo는 실행형 AI Agent로 크리에이터의 콘텐츠를 8개 언어와 중국 플랫폼으로 자동 현지화하고 배포하는 콘텐츠 글로벌화 플랫폼입니다. AI 에이전트 Robin이 번역부터 포맷 변환, 업로드까지 한 번에 처리해, 크리에이터는 콘텐츠 제작에만 집중할 수 있습니다.") | name: adobaRo, description(위 서브텍스트 문단 그대로 사용 — 별도 조합 없이 이 블록이 단일 출처), provider(Organization), areaServed(FAQ "8개 언어권" 답변의 진출 국가 목록), `hasOfferCatalog`로 요금제 페이지 Lite/Boost/Max 오퍼 연결 |
| ★★ | `Review` | 후기 3건 | ⚠️ 별점(rating) 이미지가 없으므로 `ratingValue`는 절대 임의로 넣지 말고 텍스트 리뷰로만 구성 |
| ★ | `ItemList` | 45억명/8개/1-2일/0일 통계 블록 | 선택사항. 굳이 스키마화하기보다 `<dl>`(정의 리스트)로 시맨틱 처리해도 충분 |

**비교 테이블(adobaRo 없이 vs 있을 때)**: schema.org 표준 타입이 없으므로 JSON-LD보다 `<table>` 태그로 시맨틱 마크업하는 것이 AI 파싱에 더 효과적입니다.

## 2. 기능

**헤딩 구조**
- `H1`: 글로벌 진출의 모든 과정을 하나로
- `H2`: 지금 바로 adobaRo에서 Agent에게 지시해 보세요 / 해외 플랫폼 진출을 가장 쉽게 시작하는 방법 / AI 콘텐츠 현지화로 45억 명의 글로벌 잠재 시청자에게 도달 / 글로벌 브랜드 협업 기회 확대 / 해외 플랫폼 수익을 하나로 관리 / 기존 글로벌 진출과 adobaRo의 차이점 / 자주 묻는 질문
- `H3`: 4개 지시 카테고리(글로벌 채널 개설/콘텐츠 로컬라이징/광고·브랜드 협업 매칭/수익 정산)

**스키마 마크업**
| 우선순위 | 타입 | 위치 | 비고 |
|---|---|---|---|
| ★★★ | `HowTo` | "해외 플랫폼 진출을 가장 쉽게 시작하는 방법" | step 1개(대화 한 줄로 채널 개설)라도 `HowToStep`으로 명시하면 AI 답변에 단계형으로 인용되기 좋음 |
| ★★★ | `HowTo` | 4단계 프로세스(업로드→자동 감지→로컬라이징→최적 시간 업로드) | 각 단계를 `HowToStep.text`로 |
| ★★ | `FAQPage` | 하단 FAQ 4문항 |  |
| ★ | `Service` | 4개 카테고리 각각을 `serviceType`으로 세분화해 위 Service 스키마에 연결 |  |

## 3. 요금제

**헤딩 구조**
- `H1`: 더 넓은 트래픽을 위한 완벽한 투자
- `H2`: 나만의 AI Global Agent를 채용하고 45억 명의 시청자에게 내 콘텐츠를 알리세요! (또는 이 카피는 서브헤드로 두고 "Agent 고용(월 구독형)"을 H2로) / RP(Ro-Power) 패키지(단건 구매) / RP는 이렇게 사용 돼요. / 자주 묻는 질문
- `H3`: Lite / Boost / Max, RP Package 10,000 / 30,000 / 50,000

**스키마 마크업 — 이 페이지가 GEO 임팩트 가장 큼 (가격은 AI가 즐겨 인용하는 데이터)**
| 우선순위 | 타입 | 위치 | 비고 |
|---|---|---|---|
| ★★★ | `Product` + `Offer` (플랜당 1개, 총 3개) | Lite/Boost/Max | price: 9.90/19.90/49.90, priceCurrency: USD, `billingDuration`: P1M(월 구독) |
| ★★★ | `Product` + `Offer` (패키지당 1개, 총 3개) | RP Package 10,000/30,000/50,000 | 단건 구매이므로 `priceValidUntil` 불필요, 일반 Offer로 충분 |
| ★★ | `FAQPage` | 하단 FAQ | ⚠️ **콘텐츠 이슈**: "플랜마다 Agent의 기능이 달라지나요?" 질문이 **중복 등록**되어 있음 — 스키마 넣기 전에 원본 콘텐츠부터 정리 필요 |

**RP 사용량 표**(채널 생성 5,000RP / 현지화 콘텐츠 업로드 1,000RP 등)는 스키마보다 `<table>` 시맨틱 마크업 권장.

## 4. adobaRo 소식 (이벤트/뉴스 리스팅)

**헤딩 구조**
- `H1`: 크리에이터를 위한 특별한 혜택과 최신 소식
- `H2`: 지금 adobaRo에는 이런 혜택과 소식이 있어요!
- `H3`: 카드별 이벤트명 (iQIYI 글로벌 확장 EVENT / 도우인 글로벌 확장 EVENT / Ro-Power 리워드 프로모션 / New Platform in adobaRo / NEW PLATFORM x iQIYI / 런칭 기념 EVENT) — 현재 카드 안의 텍스트가 이미지 텍스트라 실제 HTML 헤딩 태그로는 안 잡힐 가능성이 높으니, alt 텍스트가 아니라 **실텍스트 H3**로 카드 제목을 노출하는 걸 권장

**스키마 마크업**
| 우선순위 | 타입 | 위치 | 비고 |
|---|---|---|---|
| ★★★ | `Event` (카드별) | 날짜가 명시된 이벤트 (iQIYI: 2026.07.06~07.31, Ro-Power 프로모션: 2026.06.22~07.12) | startDate/endDate/eventAttendanceMode(Online)/organizer(adobaRo) |
| ★★ | `ItemList` | 전체 카드 목록을 감싸서 | itemListElement에 각 이벤트/소식 카드 순번대로 배치, 페이지네이션(1/2/3) 대응해 `ItemList.numberOfItems` 포함 |

## 5. 이벤트 상세 팝업 (iQIYI 예시)

**헤딩 구조**
- `H1` 또는 `H2`(모달이 별도 URL을 갖는지에 따라 결정): 콘텐츠를 더 넓은 세계로! iQIYI 글로벌 확장 EVENT
- `H3`: 참여 대상 / 이벤트 기간 / 리워드 지급일 같은 조건 항목들

**스키마 마크업**
- `Event`: name, startDate(2026-07-06), endDate(2026-07-31), eligibility(전체 유저 → `audience`), 리워드 내용은 `offers`(가치 있는 보상이므로)로 표현 가능

> 팝업이 SPA 오버레이라 별도 URL이 없다면, 목록 페이지의 해당 `Event` 스키마 안에 `description`으로 상세 조건을 포함시키는 게 실용적입니다.

## 6. 해외 진출 Tip (매거진 리스팅)

**헤딩 구조**
- `H1`: adobaRo 인사이트 매거진
- `H3`(리스트형이므로 H2 생략하고 카드 제목을 바로 H3로): 각 아티클 제목

📌 **참고**: 15개 카드 중 2개 외에는 아직 "제목을 입력해 주세요" 상태인데, 기획 단계라 당연한 상태로 보여요. 다만 스키마(BlogPosting)는 실제 제목·썸네일 텍스트를 값으로 넣어야 하니, 콘텐츠가 채워지는 시점에 맞춰 마크업 작업을 진행하면 됩니다.

**스키마 마크업**
| 우선순위 | 타입 | 위치 | 비고 |
|---|---|---|---|
| ★★★ | `Blog` (컬렉션) + `ItemList` | 페이지 전체 | 각 아이템에 `BlogPosting`의 요약(headline, image, url, datePublished) |
| ★ | `BreadcrumbList` | 홈 > 해외 진출 Tip |  |

## 7. 아티클 상세 페이지

이번에 업로드된 실제 아티클(틱톡·샤오홍슈·비리비리 시각적 브랜딩 편)을 기준으로:

**헤딩 구조**
- `H1`: [시각적 브랜딩] 틱톡, 샤오홍슈, 비리비리... 플랫폼이 다르면 내 채널의 '얼굴'도 달라져야 합니다
- `H2`: 핵심 요약 / 1. 도우인 & 틱톡: 1초 만에 뇌리에 꽂히는 '도파민 썸네일' / 2. 샤오홍슈(레드노트): 여백의 미를 살린 '에디토리얼 화보' / 3. 비리비리(Bilibili): 팬덤과 스토리가 얽힌 '서브컬처의 성지' / 글로벌 비주얼 아이덴티티, 아도바로로 완성하라 / 자주 묻는 질문(Q&A)
- `H3`: 각 플랫폼 섹션 내부의 "시각적 특징" / "브랜딩 가이드" — 지금은 굵은 글씨(bold) 처리로 보이는데, **H3로 승격**하면 AI가 "도우인 시각적 특징이 뭐야?" 같은 세부 질의에도 해당 블록만 정확히 뽑아 인용하기 쉬워집니다

**스키마 마크업**
| 우선순위 | 타입 | 위치 | 비고 |
|---|---|---|---|
| ★★★ | `BlogPosting`/`Article` | 페이지 전체 | headline, datePublished(2026-07-08), author(Organization: adobaRo 또는 "아도바로 팀"), image(2개 비교 썸네일), mainEntityOfPage |
| ★★★ | `FAQPage` | 하단 Q&A 2문항 | 이미 결론 우선(conclusion-first) 답변 톤이라 FAQ 스키마와 궁합이 매우 좋음 |
| ★ | `BreadcrumbList` | 홈 > 해외 진출 Tip > 이 아티클 |  |

**핵심 요약 블록** (불릿 3개)은 아티클의 TL;DR 역할을 하므로, 가능하면 `Article.abstract` 또는 `description` 필드에도 동일 텍스트를 넣어주면 AI가 전체를 읽지 않고도 요약을 그대로 인용할 확률이 높아집니다.

## 전체 우선순위 요약 (작업 순서 추천)

1. **요금제 페이지 Product/Offer** — 가격 질의에 대한 AI 인용 정확도 직결, ROI 가장 높음
2. **아티클 상세 + 매거진 리스팅의 BlogPosting/Article** — 블로그 시리즈가 이미 여러 편 발행 중이므로 템플릿화하면 반복 적용 가능
3. **전 페이지 FAQPage** — 이미 결론 우선/선언적 문체로 잘 짜여 있어 구현 난이도 대비 효과 큼
4. **소식 페이지 Event** — 날짜가 명확한 이벤트라 스키마 신뢰도 높음
5. **Main의 Service/Organization** — 브랜드 개체 인식용 기반 작업

콘텐츠가 채워지는 시점에 맞춰 함께 처리하면 좋은 항목:
- 요금제 FAQ 중복 질문 정리 (스키마에 그대로 반영되지 않도록)
- 해외 진출 Tip 리스팅 제목 확정되는 대로 BlogPosting 스키마 값 채우기
