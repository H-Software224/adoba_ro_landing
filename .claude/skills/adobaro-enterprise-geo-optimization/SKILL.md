# adobaRo Enterprise 페이지 GEO 가이드: 헤딩 구조 &amp; 스키마 마크업

> 기존 `adobaRo_GEO_스키마_헤딩_가이드.md`(B2C 7페이지)와 동일한 기준으로 작성했습니다.
> 이 페이지는 B2B용 단일 랜딩페이지(Features형)로, MCN 담당자가 "YouTube Studio 대비 무엇이 다른가"를 빠르게 파악하도록 구성되어 있습니다.

---

## 0. 전 페이지 공통 (기존 가이드와 동일하게 재사용)

| 스키마 타입 | 용도 | 비고 |
|---|---|---|
| `Organization` | 아도바 주식회사 정보 | 기존 정의 그대로 재사용 (url은 canonical root domain 고정) |
| `WebSite` | 사이트 식별 | 기존 정의 재사용 |
| `SiteNavigationElement` | 상단 GNB | 이 페이지는 로고 + 문의하기 버튼만 있어 별도 내비게이션 항목은 없음 |

## 1. 헤딩 구조 (섹션 순서대로)

| 태그 | 텍스트 | 비고 |
|---|---|---|

| (서브텍스트) | MCN을 위한 AI 채널·영상 분석 솔루션으로, 기존에 1-2주가 소요됐던 채널 분석 자료를 원클릭으로 1분만에 완성합니다 | H1 직속 설명문. 이후 `Service.description`의 소스로 사용 (기존 가이드에서 확정한 패턴과 동일) |
| `H2` | YouTube Studio를 넘어서는 MCN 전용 AI 분석 솔루션 | |
| `H3` | 원 클릭 심층 분석 / 원클릭 멀티 배포 / 방향성 제시 / 리포트 편집 | 4개 기능 카드명. 현재 굵은 텍스트로만 보이면 실제 헤딩 태그로 승격 필요 |
| `H2` | 클릭 한 번으로 생성되는 심층 분석 리포트 | |

| `H3` | YouTube 공식 API + 연동 데이터 / 자사 데이터 센터 / 채널·영상 방향성 제시 | 3단계 파이프라인 도식의 각 박스 제목 |
| `H2` | 원하는 형태로 리포트 커스터마이징 | |
| `H3` | 리포트 생성 / 영역별 설정 / 최종 편집 / 크리에이터 전달 | 4단계 프로세스 원형 스텝 |
| `H2` | YouTube Studio와 adobaRo Enterprise의 핵심 차별점 | |
| `H3` (또는 `th`) | 분석 데이터 / 소요 시간 / 리포트 주기 / 인사이트 / 커스터마이징 | 비교 테이블의 행 라벨 5개 |

## 2. 스키마 마크업 우선순위

| 우선순위 | 타입 | 위치 | 비고 |
|---|---|---|---|
| ★★★ | `Service` (또는 `SoftwareApplication`) | 페이지 전체 = adobaRo Enterprise 서비스 자체 | name: adobaRo Enterprise, description: H1 서브텍스트 그대로 사용, provider: Organization, serviceType: "MCN 채널·영상 분석 솔루션", audience: MCN(다중 채널 네트워크) |
| ★★☆ | `HowTo` | "원하는 형태로 리포트 커스터마이징" 4단계 | 리포트 생성→영역별 설정→최종 편집→크리에이터 전달을 `HowToStep`으로. 기능 페이지의 4단계 프로세스와 동일한 패턴 |
| ★★ | `ItemList` | "데이터 기반으로 제시하는 채널·영상 방향성" 3단계 파이프라인 | 사용자가 수행하는 액션이 아니라 시스템 처리 흐름이므로 `HowTo`보다 `ItemList`가 더 적합 |
| ★☆ | `ContactPoint` (Organization 하위) | 우측 상단 "문의하기" 버튼 | 버튼이 별도 문의 페이지/섹션으로 연결된다면 그 페이지에 `ContactPage`/`ContactPoint` 스키마 적용 (기존 B2C 문의하기 섹션과 동일 패턴). 단순 스크롤 이동이면 스키마 불필요 |

**비교 테이블(YouTube Studio vs adobaRo Enterprise 5개 항목)**은 schema.org 표준 타입이 없으므로, 기존 가이드 원칙대로 JSON-LD 대신 `<table>` 태그로 시맨틱 마크업하는 것을 권장합니다. (Studio 대비 데이터, 시간, 인사이트를 나란히 비교하는 부분이라 AI가 이 구조 그대로를 가장 잘 인용합니다.)

**통계 블록(1분 이내 / 약 5배)**도 스키마보다 `<dl>`(정의 리스트) 시맨틱 처리로 충분 — 기존 Main 페이지의 45억명/8개/1-2일 통계 블록과 동일한 처리 원칙.

## 3. 콘텐츠 이슈 체크 (스키마 작업 전 선행 권장)

- 4개 기능 카드(원 클릭 심층 분석 등)와 4단계 프로세스 원형(리포트 생성 등)의 제목이 실제 헤딩 태그가 아니라 카드 디자인 안의 볼드 텍스트로만 보임 → `H3`로 승격 필요
- "채널 리포트 분석 소요 시간"이라는 동일 라벨이 두 통계(1분 이내 / 약 5배)에 중복 사용됨 → 우측 값은 "심층 데이터 분량" 성격이므로 라벨 텍스트 자체를 구분해야 `H4`/`dt`로 마크업했을 때도 의미가 명확해짐
- Pricing 정보가 없는 B2B 문의형 페이지이므로 `Product`/`Offer` 스키마는 적용하지 않음 (기존 B2C 요금제 페이지와의 차이점)
